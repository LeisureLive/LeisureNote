# 每次找 AI 时的固定同步模板

这个文件用于解决两个问题：

- 每次开启新对话，不知道该怎么描述当前状态
- 长期准备过程中，容易遗漏进度、卡点和本次目标

建议原则：

- 先说当前进度，再说完成情况，再说卡点，最后说这次要我做什么
- 能少说废话就少说，优先说影响本次协作效率的信息
- 时间紧时用超短版，需要深度复盘时用完整版本

## 模板 1：日常推进版

适用于按天或按阶段推进学习任务后的日常同步。

当前默认有两条主线：

- 技术主线：
  - `prepare/2026/tracks/xiaomi_java_data_service/01_1month_plan.md`
- 算法专项：
  - `prepare/2026/tracks/xiaomi_java_data_service/algorithms/90_1month_algorithm_plan.md`

```md
我当前在推进：
- 技术主线 / 算法专项 / 两者都在推进

## 当前进度
- 技术主线当前进度：
- 算法专项当前进度：
- 今天完成了：
- 今天没完成的：
- 当前最卡的点：

## 我的掌握情况
- 我已经能讲清的内容：
- 我讲不顺 / 不理解的内容：

## 今日产出
- 我整理的答案提纲：
- 我做的算法题：
- 我新增的薄弱点：

## 这次希望你帮我做什么
- 例如：复盘技术主线 Day 3
- 复盘算法专项 Day 10
- 帮我讲清某个知识点
- 帮我检查某个回答
- 帮我继续推进到下一天
```

## 模板 3：知识点复盘版

适用于某个知识点“看过了，但不会答”或“答得不稳”的情况。

```md
我现在进行到：
当前复习主题：
- 所属主线：
  - 技术主线 / 算法专项

我的状态：
- 我已经理解的部分：
- 我容易混淆的部分：
- 我现在属于：
  - 看过但讲不出来 / 知道结论但不懂原理 / 原理懂了但不会答题

这次要你帮我：
- 用面试答题方式帮我梳理
- 给我高频追问
- 帮我判断我该掌握到什么深度
```

## 推荐的最小描述规则

如果你不想每次都写很多，至少告诉我下面 4 件事：

- 我现在技术主线 / 算法专项分别到哪
- 我已经完成了什么
- 我当前卡在哪
- 这次你要我做什么

最小示例：

```md
技术主线：第 2 周 Day 10
算法专项：Week2 Day12
今天完成了：Spring 事务、Spring MVC 主流程
当前卡点：事务失效场景总讲不清
这次要你帮我：按面试答题方式帮我复盘，并最后追问我 3 个问题
```
