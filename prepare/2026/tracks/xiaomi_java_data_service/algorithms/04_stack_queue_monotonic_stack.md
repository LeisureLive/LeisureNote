# 算法专题 04：栈 / 队列 / 单调栈

适用目标：

- 理解 `LIFO` 和 `FIFO` 两种结构分别适合什么问题
- 建立“括号匹配、最近更大 / 更小元素、区间展开”这几类高频模式
- 避免把单调栈题误写成暴力双重循环

## 1. 栈和队列的核心特点

- 栈：后进先出，最适合处理“最近一次状态”
- 队列：先进先出，最适合处理“按顺序流入流出”的场景
- 单调栈：栈内元素保持单调，适合快速找到“左边 / 右边第一个更大或更小的元素”
- 很多题表面看是数组题，本质上是“历史状态如何被压缩保存”

这里的“单调栈”先要把定义说清楚：

- 单调栈不是一种新数据结构，本质上还是普通栈
- 只是额外要求：
  - 栈中保存的元素（通常是下标对应的值）要始终满足某种单调关系
- 为了避免歧义，后面默认都按“栈底 -> 栈顶”的顺序描述递增 / 递减

也就是说：

- 递增栈：
  - 指栈底到栈顶对应的值越来越大，或者至少不变
  - 栈顶通常是当前这组候选里较大的那个
- 递减栈：
  - 指栈底到栈顶对应的值越来越小，或者至少不变
  - 栈顶通常是当前这组候选里较小的那个

举例：

- 如果栈里下标对应的值是 `[2, 4, 7]`，这是递增栈
- 如果栈里下标对应的值是 `[9, 6, 3]`，这是递减栈

补充一个最容易混的点：

- 很多人说“维护递增栈”，结果脑子里想的是“当前元素越来越大就入栈”
- 真正该看的不是入栈顺序，而是“入栈完成后，栈底到栈顶的值是否保持递增 / 递减”
- 只要当前元素会破坏这种顺序，就要先弹栈，再把当前元素压进去

## 2. 什么题应该优先想到栈 / 队列

如果题目有下面这些信号，通常优先考虑：

- “括号是否匹配”
- “表达式求值”
- “最近一个更大 / 更小元素”
- “柱状图面积”
- “滑动窗口最大值”
- “按到达顺序处理任务”

## 3. 常考模式

看这一节时，建议你每个模式都先问自己三件事：

- 结构里到底存“值”还是“下标”
- 什么时候入栈 / 入队，什么时候出栈 / 出队
- 当前结构是在保存历史状态，还是在维护某种单调性

### 3.1 普通栈：匹配、回退、延迟处理

适用场景：

- 括号匹配
- 逆波兰表达式
- 路径化简

代表题：

- `20. 有效的括号`
- `150. 逆波兰表达式求值`
- `71. 简化路径`

核心理解：

- 栈保存的是“后面还要回来用”的上下文
- 不是所有入栈出栈都叫栈题，关键是当前处理依赖最近未消化状态

通用套路：

1. 先判断当前问题是不是依赖“最近还没处理完的信息”。
2. 如果是，就考虑用栈把这些未消化状态暂存起来。
3. 常见主流程是：
   - 遇到新信息时先决定要不要入栈
   - 如果当前信息能消化栈顶状态，就不断出栈处理
   - 最后检查栈是否处于合法结束状态
4. 普通栈最适合做：
   - 匹配
   - 回退
   - 延迟结算

你真正要会的是：

- 不是只会“push / pop”
- 而是知道栈里保存的是哪一类未完成上下文

### 3.2 队列：顺序处理与窗口推进

适用场景：

- 按顺序消费任务
- 层序遍历
- 滑动窗口中的候选下标维护

代表题：

- `232. 用栈实现队列`
- `225. 用队列实现栈`
- `239. 滑动窗口最大值`

通用套路：

1. 先判断题目是不是强调“先来先处理”或“按层 / 按时间顺序推进”。
2. 如果核心是顺序流入流出，就优先考虑队列。
3. 常见主流程是：
   - 新元素从队尾进入
   - 过期元素从队头移出
   - 队头通常代表当前最早或当前最该处理的状态
4. 如果题目还要求窗口内最值，那普通队列不够，通常要升级为单调队列。

你真正要会的是：

- 能区分“普通顺序处理”和“窗口最值维护”
- 知道什么时候普通队列就够，什么时候要更强的队列结构

### 3.3 单调栈：找最近更大 / 更小元素

这一节只抓 3 件事：

1. 什么信号说明要用单调栈  
2. 单调栈里的“递增 / 递减”到底指什么  
3. 左 / 右第一个更大 / 更小元素怎么快速判断模板

#### 1. 使用信号

如果题目出现下面这些表达，优先考虑单调栈：

- 每个位置都要找：
  - 左边第一个更大 / 更小
  - 右边第一个更大 / 更小
- 题目强调：
  - 最近一个
  - 第一个满足条件的元素
  - 不想写暴力双重循环
- 你发现自己在做的其实是：
  - 对每个元素向左或向右一直扫，直到遇到第一个满足条件的值

典型代表题：

- `739. 每日温度`
- `496. 下一个更大元素 I`
- `1475. 商品折扣后的最终价格`

#### 2. 单调栈的“单调”指什么

- 单调栈本质上还是普通栈，不是新数据结构
- 所谓“单调”，指的是：
  - 栈里这些元素对应的值，始终保持单调关系
- 后文统一约定：
  - 都按“栈底 -> 栈顶”的顺序描述递增 / 递减

所以：

- 递增栈：
  - 栈底到栈顶的值递增
  - 例如 `[2, 4, 7]`
- 递减栈：
  - 栈底到栈顶的值递减
  - 例如 `[9, 6, 3]`

补一句最容易混的点：

- “维护递增栈”不是说“当前元素越来越大就直接入栈”
- 真正要看的是：
  - 入栈完成后，栈底到栈顶还能不能保持递增 / 递减
- 如果当前元素会破坏这个顺序，就要先弹栈，再入栈

#### 3. 左 / 右第一个更大 / 更小元素的通用套路

先记住这 4 句话：

- “找右边”时，当前元素是在解决别人
- “找左边”时，当前元素是在查询自己
- “更大 / 更小”决定栈的单调方向
- “是否带等号”决定 `while` 里比较符号要不要包含 `=`

再记住统一习惯：

- 基础题默认都可以先按“从左到右”遍历
- 栈里通常存下标，不存值
  - 因为后面往往还要回填答案，或者还要用位置信息

最实用的判断表如下：

| 目标 | 怎么遍历 | 用什么单调栈 | 关键条件 |
| --- | --- | --- | --- |
| 右边第一个严格更大 | 从左到右 | 递减栈 | `nums[i] > nums[stack.peek()]` |
| 右边第一个大于等于 | 从左到右 | 严格递减栈 | `nums[i] >= nums[stack.peek()]` |
| 右边第一个严格更小 | 从左到右 | 递增栈 | `nums[i] < nums[stack.peek()]` |
| 右边第一个小于等于 | 从左到右 | 严格递增栈 | `nums[i] <= nums[stack.peek()]` |
| 左边第一个严格更大 | 从左到右 | 递减栈 | 先弹 `<= nums[i]` |
| 左边第一个大于等于 | 从左到右 | 严格递减栈 | 先弹 `< nums[i]` |
| 左边第一个严格更小 | 从左到右 | 递增栈 | 先弹 `>= nums[i]` |
| 左边第一个小于等于 | 从左到右 | 严格递增栈 | 先弹 `> nums[i]` |

可以把它压成两句更短的判断：

- 想找“更大”，就维护递减栈
- 想找“更小”，就维护递增栈

然后再分左右：

- 找右边：
  - 当前元素出现时，用它去弹栈并结算前面元素
- 找左边：
  - 当前元素先把不合格候选弹掉，然后直接看栈顶

对应的两个通用骨架如下。

找右边第一个满足条件的元素：

```java
for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && 当前元素可以结算栈顶) {
        ans[stack.pop()] = i;
    }
    stack.push(i);
}
```

找左边第一个满足条件的元素：

```java
for (int i = 0; i < n; i++) {
    while (!stack.isEmpty() && 栈顶不可能成为当前元素答案) {
        stack.pop();
    }
    ans[i] = stack.isEmpty() ? -1 : stack.peek();
    stack.push(i);
}
```

### 3.4 单调栈求区间边界

适用场景：

- 需要求某个元素作为最小值 / 最大值时能影响的区间
- 需要把二维或区间问题压成“边界扩展”问题

代表题：

- `84. 柱状图中最大的矩形`
- `42. 接雨水`
- `85. 最大矩形`

通用套路：

1. 先识别题目是不是在问“某个元素向左右最远能扩到哪里”。
2. 这类题常见思路是：
   - 找左边界
   - 找右边界
   - 再用边界计算面积、宽度或贡献
3. 常见主流程是：
   - 维护单调栈保存候选边界
   - 一旦当前元素让栈顶元素失去扩展资格，就弹栈
   - 弹栈那一刻顺手结算以它为核心的答案
4. 这类题经常需要在数组末尾补一个哨兵，帮助把栈清空。

你真正要会的是：

- 能看出这是“边界题”而不只是“最近更大 / 更小题”
- 能说清弹栈时左右边界分别是谁

## 4. 栈 / 队列题的高频坑点

- 栈里该存“值”还是“下标”没想清楚
- 单调栈中“严格大于”和“大于等于”条件写混
- “找左边”和“找右边”的写法混在一起，导致答案回填对象错位
- 不清楚当前是闭区间思维还是“当前元素正在结算历史元素”的思维，结果遍历和弹栈顺序写乱
- 忘记处理遍历结束后栈里剩余元素
- 队列题只会用普通队列，却意识不到需要单调队列
- 层序遍历时把当前层和下一层节点混在一起

## 5. 栈 / 队列题的面试表达模板

```text
这题本质是在找每个位置左边或右边第一个更大 / 更小元素，不适合暴力双重循环。
我会先判断它是在找左边还是右边，再判断是在找更大还是更小。
如果是找右边，当前元素通常是在结算前面栈里那些还没出答案的元素；
如果是找左边，当前元素通常是在清掉不合格候选后，直接从栈顶查询自己的答案。
栈里我通常存下标，因为后面往往还需要位置信息或回填答案。
每个元素最多进栈出栈一次，所以整体复杂度通常是 O(n)。
```

## 7. 你复习这块时最低要做到什么程度

- 能区分什么时候该用普通栈，什么时候该用单调栈
- 能说清单调栈里为什么常存下标
- 能说清“每个元素最多进栈出栈一次，所以是 O(n)”
- 能独立写出 `20` 和 `739` 这两类最基础模板

## 8. 本专题练习规划

### 第一阶段：模板入门

1. `20. 有效的括号`
   - 训练重点：
     - 普通栈保存未消化状态
     - 入栈 / 出栈匹配规则
2. `739. 每日温度`
   - 训练重点：
     - 单调栈基础模板
     - 栈里为什么存下标

### 第二阶段：高频主战

1. `496. 下一个更大元素 I`
   - 训练重点：
     - 最近更大元素模板
     - 单调栈如何配合哈希表输出答案
2. `239. 滑动窗口最大值`
   - 训练重点：
     - 单调队列
     - 区分“普通队列”和“维护最值的队列”
3. `84. 柱状图中最大的矩形`
   - 训练重点：
     - 单调栈求左右边界
     - 训练“弹栈时结算答案”的思维

### 第三阶段：变体 / 验收

1. `42. 接雨水`
   - 训练重点：
     - 同一道题可从双指针 / 单调栈角度理解
     - 训练跨专题迁移能力
2. `85. 最大矩形`
   - 训练重点：
     - 把二维问题转成多次 `84`
     - 检查你是否真的理解了单调栈边界模型

## 9. 专题验收标准

  - 普通栈
