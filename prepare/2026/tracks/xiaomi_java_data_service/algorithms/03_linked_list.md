# 算法专题 03：链表

适用目标：

- 理解链表题为什么本质上是“指针操作题”
- 建立 `dummy node`、快慢指针、反转、合并四条主线
- 减少空指针、断链、头节点变化这类低级错误

## 1. 链表的核心特点

- 物理内存不要求连续
- 按下标随机访问不方便，通常不能像数组一样 `O(1)` 取第 `k` 个元素
- 已知前驱节点时，插入 / 删除节点通常比数组更自然
- 很多题的关键不是“值怎么算”，而是“指针怎么改”
- 一旦指针断掉，后续链路就容易丢失，所以操作顺序非常重要

## 2. 什么题应该优先想到链表

如果题目有下面这些信号，通常优先考虑链表思维：

- 输入结构是 `ListNode`
- 题目要求：
  - 反转链表
  - 删除某个节点
  - 合并两个链表
  - 找倒数第 `k` 个节点
  - 判断是否有环
- 需要原地调整节点顺序，而不是新建大量额外结构
- 题目重点在“前驱 / 后继关系”而不是“随机访问”

## 3. 链表题常考模式

看这一节时，建议你每个模式都固定追问三件事：

- 这题到底在改哪几个指针关系
- 哪个节点可能变化，尤其是头节点会不会变
- 为了不丢链，代码里第一步应该先保存什么

### 3.1 虚拟头节点 `dummy node`

适用场景：

- 删除头节点
- 合并链表
- 分段拼接
- 需要统一处理“第一个节点”和“中间节点”

代表题：

- `21. 合并两个有序链表`
- `19. 删除链表的倒数第 N 个结点`
- `203. 移除链表元素`

为什么常用：

- 它能把“头节点要不要特殊处理”这个分支消掉
- 让代码更稳定，也更不容易漏边界

通用套路：

1. 先判断题目会不会碰到头节点变化。
2. 如果头节点也可能被删除、插入或重新拼接，就优先引入 `dummy node`。
3. 常见主流程是：
   - 创建 `dummy.next = head`
   - 用一个指针从 `dummy` 出发处理链表
   - 最后统一返回 `dummy.next`
4. 这样做的价值是把“第一个节点”和“中间节点”统一成同一种处理方式。

你真正要会的是：

- 不是把 `dummy node` 当口诀背
- 而是知道它本质上是在统一前驱节点

### 3.2 快慢指针

适用场景：

- 找中点
- 找倒数第 `k` 个节点
- 判断是否有环
- 判断环入口

代表题：

- `141. 环形链表`
- `142. 环形链表 II`
- `876. 链表的中间结点`

核心理解：

- 不同速度的指针，本质是在制造“相对位移”
- 不是只会背“快两步慢一步”，而是要知道这个速度差在解决什么问题

通用套路：

1. 先明确这题要找的是什么：
   - 中点
   - 倒数第 `k` 个节点
   - 是否有环
   - 环入口
2. 再决定快慢指针之间要制造哪种距离关系。
3. 常见主流程有两类：
   - 同时出发，靠速度差相遇或定位
   - 先让快指针先走若干步，再同步前进
4. 写代码前一定先定好循环条件，不然最容易空指针。

你真正要会的是：

- 每种快慢指针都能说清“速度差 / 间距”在解决什么问题
- 而不是统一背成一个模板

### 3.3 原地反转

适用场景：

- 整体反转
- 区间反转
- `k` 个一组反转

代表题：

- `206. 反转链表`
- `92. 反转链表 II`
- `25. K 个一组翻转链表`

核心理解：

- 反转的本质是让当前节点的 `next` 指回前驱节点
- 做这类题时，最关键的是先保存 `next`，再改指针

通用套路：

1. 先固定三件事：
   - `prev` 指向谁
   - `curr` 指向谁
   - `next` 用来临时保存谁
2. 常见主流程是：
   - 先保存 `next`
   - 再改 `curr.next`
   - 然后整体向前推进
3. 如果是区间反转或分组反转，本质上也是先局部反转，再把前后段重新接回去。
4. 反转题最怕的不是不会写，而是顺序写错导致断链。

你真正要会的是：

- 能脱稿说出“先存 next，再改指针，再推进”的顺序为什么不能反
- 能区分整体反转和局部反转只是外层组织不同

### 3.4 合并与重排

适用场景：

- 合并两个有序链表
- 重新拼接奇偶位
- 链表重排

代表题：

- `21. 合并两个有序链表`
- `328. 奇偶链表`
- `143. 重排链表`

通用套路：

1. 先判断这题是在做：
   - 两条链的比较合并
   - 一条链的拆分再拼接
   - 多段链表的重排
2. 如果是合并两条有序链，常见主流程是：
   - 比较两个当前节点
   - 取较小者接到结果链尾部
   - 推进对应链表
3. 如果是重排题，通常先拆成几个子问题：
   - 找中点
   - 反转后半段
   - 交替合并
4. 这类题的关键不是某一步很难，而是每一步之间的衔接别断链。

你真正要会的是：

- 先把复杂题拆成几个基础链表动作
- 再按顺序组合，而不是整题一起硬写

### 3.5 链表 + 哈希表

适用场景：

- 需要快速判断节点是否访问过
- 需要缓存节点位置或前驱关系

代表题：

- `138. 随机链表的复制`
- `146. LRU 缓存`

通用套路：

1. 先判断链表题里是不是还需要“快速查历史节点信息”。
2. 如果需要通过旧节点快速找到：
   - 新节点
   - 前驱关系
   - 是否访问过
   那通常要引入哈希表。
3. 常见主流程是：
   - 第一轮遍历建立映射
   - 第二轮遍历补全指针关系
   - 或一边遍历一边查询历史节点
4. 这类题本质上是在用哈希表补链表“随机访问差”的短板。

你真正要会的是：

- 能判断什么时候纯指针不够，必须上哈希表
- 能说清 map 里到底在存“旧节点到新节点”还是“节点是否访问过”

## 4. 链表题的高频坑点

- 忘记保存 `next`，改完指针后链路丢失
- 头节点变化后还返回旧头节点
- 删除节点时没有拿到前驱节点
- 快慢指针边界没处理好，导致空指针
- 写出了思路，但没有统一“谁是当前节点、谁是前驱、谁是后继”

## 5. 链表题的面试表达模板

```text
这题输入是链表，所以我会先从指针关系而不是下标访问去想。
如果题目涉及删除头节点或者统一拼接逻辑，我会先引入 dummy node；
如果题目涉及中点、倒数第 k 个节点或环，我会优先考虑快慢指针。
接下来我会先定义当前节点、前驱节点和后继节点分别是谁，
再说明每一步指针如何移动，最后补复杂度和边界。
```

## 7. 你复习链表时最低要做到什么程度

- 能说清 `dummy node` 在解决什么问题
- 能把快慢指针的典型用途区分开
- 能脱稿讲清“反转链表”每一步指针怎么变
- 遇到链表题时，先画出节点关系再写代码，而不是直接硬写

## 8. 本专题练习规划

### 第一阶段：模板入门

1. `206. 反转链表`
   - 训练重点：
     - 保存 `next`
     - 原地改指针
2. `21. 合并两个有序链表`
   - 训练重点：
     - `dummy node`
     - 指针推进和尾部拼接

### 第二阶段：高频主战

1. `19. 删除链表的倒数第 N 个结点`
   - 训练重点：
     - 快慢指针制造间距
     - 删除节点时为什么常配合 `dummy node`
2. `141. 环形链表`
   - 训练重点：
     - 快慢指针判环
     - 为什么快慢指针一定会相遇
3. `143. 重排链表`
   - 训练重点：
     - 找中点 + 反转后半段 + 合并
     - 训练链表题常见组合能力

### 第三阶段：变体 / 验收

1. `142. 环形链表 II`
   - 训练重点：
     - 判环升级到找环入口
     - 训练推导而不是死记结论
2. `25. K 个一组翻转链表`
   - 训练重点：
     - 分组处理
     - 多段链表反转与拼接

## 9. 专题验收标准

- 能在纸上画清 `206`、`19`、`141` 的指针变化
- 能讲清 `dummy node`、快慢指针、反转这三类模板分别在解决什么问题
- 能做到写链表题前先定义：
  - 当前节点
  - 前驱节点
  - 后继节点
- 遇到链表题时，不再第一反应硬写，而是先判断它属于哪条主线
