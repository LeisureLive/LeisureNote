# 算法专题 12：贪心

适用目标：

- 建立贪心题的统一认知：贪心的关键不是“拍脑袋选一个当前最优”，而是先说明这个局部选择为什么会让后续更优或不变差
- 分清覆盖范围型、区间选择型、维护历史最优状态型，以及“贪心 + 二分 / 堆”这几类高频模式
- 能把贪心题从“凭感觉写”提升到“能解释贪心依据、状态含义和为什么正确”
- 减少贪心题最常见的局部最优依据不清、排序规则写错、把贪心题误判成 DP 或 BFS 的问题

## 1. 贪心的核心特点

- 贪心适合的问题通常有一个共同点：
  - 当前这一步可以做一个局部最优选择，并且这个选择不会破坏全局最优答案
- 直白一点说，就是：
  - 你不需要穷举所有可能
  - 也不需要完整保存所有历史状态
  - 只要在每一步做对当前最合适的选择，就能把答案推进出来
- 贪心最重要的不是“会不会猜”，而是先看清下面三件事：
  - 当前局部最优依据是什么
  - 为什么这个依据不会让后面更差
  - 这题到底是在求：
    - 可达性
    - 最少次数
    - 最大收益
    - 区间选择最优
- 真正决定贪心能不能写对的，通常不是代码量，而是：
  - 你有没有找对“每一步该贪什么”
  - 你知不知道这个贪心依据为什么成立

## 2. 什么题应该优先想到贪心

如果题目有下面这些信号，通常优先考虑贪心：

- 当前只需要维护一个最优边界或最优候选：
  - 当前最远能跳到哪里
  - 当前最小买入价格
  - 当前区间的最晚结束点
- 每一步的选择都只影响“后续空间”，而不是要求列出所有方案
- 题目问的是：
  - 能不能到终点
  - 最少几次操作
  - 最大收益
  - 最多能选多少个不冲突区间
- 排序后，局部比较规则非常清晰：
  - 谁结束更早
  - 谁起点更晚
  - 谁当前收益更优
- 可以把问题描述成：
  - 同样长度 / 数量下，让结尾更小更优
  - 同样能覆盖当前位置时，让下一步覆盖更远更优

如果题目要求枚举所有方案，更可能是回溯；如果题目要求最优值且状态需要复用，很多时候更像 DP；如果题目在无权图上求最少步数，很多时候先考虑 BFS。

## 3. 贪心题常考模式

看这一节时，建议你每个模式都固定追问三件事：

- 当前到底在“贪”什么
- 这个局部选择为什么对后续更有利
- 结果是一步步直接产生，还是需要额外统计答案

### 3.1 覆盖范围 / 可达性：每一步尽量把边界推得更远

适用场景：

- 能否到达终点
- 最少跳几次到终点
- 当前选择会影响后面最远可覆盖范围

代表题：

- `55. 跳跃游戏`
- `45. 跳跃游戏 II`

核心理解：

- 这类题的重点不是枚举每一种跳法，而是维护“当前最远能覆盖到哪里”
- `55` 的重点是：
  - 只要当前位置在当前可覆盖范围内，就继续更新最远覆盖边界
  - 一旦某个位置已经超出可覆盖范围，就说明到不了了
- `45` 的重点是：
  - 在当前这一步能覆盖到的区间内，顺手计算下一步最远能覆盖到哪里
  - 当前区间扫完时，就必须做一次跳跃

通用套路：

1. 先维护一个当前可达边界。
2. 遍历当前位置时，用它更新下一步最远可达位置。
3. 如果当前位置已经超过当前边界，说明失败。
4. 如果是在求最少跳数，就把每一层可达区间看成“一次跳跃能覆盖的范围”。

你真正要会的是：

- 能说清 `55` 为什么不需要回头枚举跳法
- 能说清 `45` 为什么本质上是在按区间分层，而不是暴力试每个起点

### 3.2 维护历史最优状态：当前决策依赖一个最优历史值

适用场景：

- 股票买卖
- 当前收益由“当前值 - 历史最优基准”决定
- 每一步只需要保留一个历史最好候选

代表题：

- `121. 买卖股票的最佳时机`
- `122. 买卖股票的最佳时机 II`

核心理解：

- `121` 的重点是：
  - 遍历到今天时，最大利润只取决于：
    - 今天卖出能赚多少
    - 以及此前最便宜的买入价格是多少
- 所以你不用记住所有历史价格，只要维护：
  - `minPrice`
  - `maxProfit`

通用套路：

1. 先定义当前答案依赖的历史最优状态。
2. 一边遍历，一边更新这个历史状态。
3. 再用当前值和历史最优状态更新答案。

你真正要会的是：

- 能说清这类题为什么不需要 DP 数组
- 能说清“维护一个历史最优值”本身就是贪心

### 3.3 区间选择 / 区间覆盖：排序后做局部最优选择

适用场景：

- 选最多不重叠区间
- 删最少区间使其不重叠
- 用最少资源覆盖所有区间

代表题：

- `435. 无重叠区间`
- `452. 用最少数量的箭引爆气球`

核心理解：

- 这类题的关键往往不是区间本身，而是排序规则
- 最常见、最值得优先想到的规则是：
  - 按结束位置排序
- 为什么：
  - 结束越早，给后面留下的可选择空间越大
- `435` 的重点是：
  - 想保留尽可能多的不重叠区间，本质上等价于删除最少区间
- `452` 的重点是：
  - 当前箭能覆盖的公共重叠区间尽量继续缩
  - 一旦不重叠，就必须新开一支箭

通用套路：

1. 先按结束位置排序，或按题意找到最合理的排序键。
2. 维护当前已经选择的最后边界或当前重叠区间。
3. 新区间与当前边界冲突时，决定：
   - 跳过
   - 替换
   - 新开一次操作

你真正要会的是：

- 能说清为什么“按结束时间早的优先”是对后续最友好的
- 能区分“最多保留几个”和“最少删除几个”其实是同一条主线

### 3.4 贪心 + 二分 / 数据结构：局部最优信息需要高效维护

适用场景：

- 同长度下，让结尾尽量更小
- 局部最优候选需要快速更新
- 堆或二分只是加速手段，核心仍是贪心选择依据

代表题：

- `300. 最长递增子序列`
- `871. 最低加油次数`

核心理解：

- `300` 的优化思路里，贪心核心是：
  - 在长度相同的递增子序列里，结尾越小越好
  - 结尾越小，后面越容易接上更大的数
- 二分不是核心思想，只是用来更快找到应该替换哪个结尾位置

你真正要会的是：

- 能区分“贪心依据”和“二分 / 堆只是实现手段”
- 能把 `300` 说成“DP 可做，贪心 + 二分可优化”

## 4. 贪心题的统一判断框架

做贪心题时，可以先按这 5 步判断：

1. 当前一步到底在决定什么：
   - 跳到哪里
   - 买或卖
   - 选或不选某个区间
2. 当前局部最优依据是什么：
   - 覆盖更远
   - 价格更低
   - 结束更早
   - 结尾更小
3. 为什么这个局部选择不会让未来更差：
   - 是否能给后面留下更大空间
4. 答案是实时产生，还是最后统计：
   - 例如最大利润
   - 或最少操作次数
5. 是否需要排序、二分或堆来辅助：
   - 排序是为了让局部比较规则稳定
   - 二分 / 堆是为了更快维护最优候选

如果这 5 件事说不清，贪心题通常会写成“能过样例，但解释不清为什么对”。

## 5. 贪心题的高频坑点

- 上来就说“感觉贪心可以”，但说不出贪心依据
- 把“当前看起来最大”误当成真正的局部最优
- 区间题排序规则写错：
  - 该按结束排序，却按开始排序
- 把 `45` 这类题误写成 BFS 或 DP，复杂度过高
- 把 `121` 这类题写成完整 DP，虽然能做，但没有抓住题目本质
- `300` 优化解里把 `tails` 误认为真实 LIS 路径
- 严格递增条件下，把 `>` 写成 `>=`

## 6. 贪心题的面试表达模板

```text
这题我会先看当前每一步是不是存在一个稳定的局部最优选择，
并且这个选择会让后续空间更大或者至少不变差。

如果有，我会优先考虑贪心，而不是先上 DP 或暴力搜索。

接下来我会明确：
1. 当前到底在贪什么
2. 为什么这个局部选择是合理的
3. 结果是实时更新，还是需要最后统计

如果题目涉及区间，我会先看排序规则；
如果题目涉及“同长度下结尾更优”，我会考虑贪心 + 二分；
如果题目只需要维护一个历史最优值，我会考虑单次遍历直接更新答案。
```

## 8. 你复习贪心时最低要做到什么程度

- 能区分：
  - 覆盖范围型
  - 历史最优状态型
  - 区间排序型
  - 贪心 + 二分
- 能独立写出：
  - `55`
  - `121`
  - `435`
- 能讲清：
  - 当前贪心依据是什么
  - 为什么这个局部选择不会让未来更差
  - `300` 为什么既能用 DP，也能用贪心 + 二分优化
- 口头表达时，先说：
  - 当前局部选择是什么
  - 为什么成立
  - 需要维护什么边界或历史状态
