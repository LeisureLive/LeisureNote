# 1 个月算法专项计划

适用范围：

- 这是一份独立于技术知识复习主线的算法专项计划
- 从当前进度继续推进，不再强绑定 `01_1month_plan.md` 里的日程
- 目标是在 1 个月内，把常见数据结构和算法模式练到“面试可用”

## 1. 目标与边界

### 目标

这 1 个月的算法目标不是“刷很多题”，而是做到下面 4 件事：

1. 常见专题都过一遍：
   - 数组
   - 哈希表
   - 链表
   - 栈 / 队列 / 单调栈
   - 字符串
   - 二叉树 / DFS / BFS
   - 堆 / 优先队列
   - 二分查找
   - 回溯
   - 图
   - 动态规划
2. 每个专题都至少拿稳：
   - 1 道模板题
   - 1-2 道高频题
   - 1 道变体或验收题
3. 不只会写代码，还要能讲：
   - 为什么想到这个方法
   - 状态怎么定义
   - 为什么复杂度成立
   - 这题和同专题其他题的区别
4. 建立自己的“题型识别 -> 模板调用 -> 口头表达”节奏

### 边界

即使增加强度，1 个月里“吃透”的合理含义也应该是：

- 吃透各专题的高频模板和常见面试题
- 能完成中低难度高频题
- 遇到同类题时能迁移

而不是：

- 所有专题所有变体都完全掌握
- 高难 DP / 图论 / 复杂构造都稳定秒出

## 2. 时间强度建议

为了在 1 个月内覆盖全部专题，建议算法专项至少投入下面的额外时间：

- 工作日：
  - 每天 `1.5h`
- 周末：
  - 每天 `3h`
- 每周总算法时间：
  - 约 `13.5h`

如果低于这个投入：

- 可以继续推进
- 但月内很难把所有专题都过到“面试可用”

如果目标进一步提高到“从 Week2 开始就尽量吃透专题重点”，建议从 `Week2` 开始把强度提升到：

- 工作日：
  - 每天 `2h`
- 周末：
  - 每天 `4-5h`

原因：

- `Week2-Week4` 不再只是做题，还要同步完成专题总结和专题验收
- `Week2` 虽然偏基础模板，但正适合在这一周把专题框架打牢
- `Week3-Week4` 覆盖的是树 / 堆 / 字符串进阶 / 回溯 / 图 / DP
- 这些专题如果只有“做题时间”，没有“专题总结时间”，很容易停留在会做题但讲不清

## 3. 每日训练固定动作

### 3.1 工作日标准版（90 分钟）

1. `10-15` 分钟：
   - 复盘前一天题目
   - 口头讲 1 道旧题
2. `35-40` 分钟：
   - 做当天第 1 题
3. `25-30` 分钟：
   - 做当天第 2 题或变体
4. `10-15` 分钟：
   - 写本题关键信号、状态定义、易错点

### 3.2 周末标准版（180 分钟）

1. `20-30` 分钟：
   - 复盘本周模板题
2. `60-80` 分钟：
   - 做 1 道高频主战题
3. `40-50` 分钟：
   - 做 1 道验收题 / 变体题
4. `20-30` 分钟：
   - 口头讲 2 题
   - 更新错题和薄弱点

### 3.3 Week2-Week4 强化版（120 分钟工作日）

从 `Week2` 开始，如果目标是尽量吃透专题，建议工作日按下面结构执行：

1. `15-20` 分钟：
   - 复盘前一天专题模板
   - 口头讲 1 道旧题
2. `45-50` 分钟：
   - 做当天主做题
3. `30-35` 分钟：
   - 做当天加练题
4. `15-20` 分钟：
   - 写专题总结：
     - 题型信号
     - 状态定义
     - 易错点
     - 代表题之间的区别

### 3.4 Week2-Week4 强化版（周末 240-300 分钟）

1. `30-40` 分钟：
   - 回看本周专题框架
2. `70-90` 分钟：
   - 主做 1 道高频主战题
3. `50-60` 分钟：
   - 做 1 道变体 / 验收题
4. `30-40` 分钟：
   - 口头讲 2-3 题
5. `30-40` 分钟：
   - 输出专题总结或错题归档
