# 算法专题 05：字符串

适用目标：

- 建立字符串题的基础认知：它本质上往往是数组 + 指针 + 哈希的组合
- 区分“子串、子序列、前缀、后缀、回文”这些常见概念
- 避免因为定义不清导致题型识别错误

## 1. 字符串题的核心特点

- 字符串可以看成特殊数组，但更强调字符语义和边界定义
- 很多题的关键不是数据结构本身，而是：
  - 如何维护连续区间
  - 如何判断字符是否重复
  - 如何构造稳定的比较规则
- Java 里字符串不可变，所以频繁拼接时要注意 `StringBuilder`

## 2. 什么题应该优先想到字符串

如果题目有下面这些信号，通常优先考虑字符串思维：

- “最长 / 最短子串”
- “是否为回文”
- “字符串匹配”
- “按字符统计频次”
- “字符串解析 / 模拟”
- “字符串重组 / 反转 / 分组”

## 3. 字符串题常考模式

看这一节时，建议你不要只盯着“字符串”两个字，而要固定追问三件事：

- 题目到底在考连续区间、字符关系，还是字符串解析
- 状态是按字符频次维护，还是按左右指针维护
- 这题本质更像数组、哈希还是模拟

### 3.1 滑动窗口

适用场景：

- 最长无重复子串
- 包含目标字符集合的最短子串
- 找所有异位词起点

代表题：

- `3. 无重复字符的最长子串`
- `438. 找到字符串中所有字母异位词`
- `76. 最小覆盖子串`

关键点：

- 一定要先分清“窗口是否合法”的定义
- 再决定左指针什么时候缩、右指针什么时候扩

通用套路：

1. 先确认这是子串问题，而且窗口必须连续。
2. 再定义窗口状态：
   - 字符是否重复
   - 目标字符是否覆盖
   - 频次是否匹配
3. 常见主流程是：
   - 右指针扩张，把新字符纳入窗口
   - 更新计数或窗口状态
   - 当窗口满足或不满足条件时，移动左指针调整
   - 在合适时机更新答案
4. 字符串窗口题最容易错的，不是代码，而是“窗口合法”定义没讲清。

你真正要会的是：

- 先讲清窗口在维护什么
- 再讲左右指针为什么这样动

### 3.2 双指针 / 回文判断

适用场景：

- 判断回文串
- 原地反转字符数组
- 从两端向中间收缩

代表题：

- `125. 验证回文串`
- `344. 反转字符串`
- `680. 验证回文串 II`

通用套路：

1. 先判断题目是不是在比较两端字符关系。
2. 如果是，就优先考虑双指针从两端向中间逼近。
3. 常见主流程是：
   - 左右指针分别从两端出发
   - 跳过无效字符或先处理大小写
   - 比较当前字符
   - 相同就继续收缩，不同就按题意返回或尝试补救
4. 这类题的关键通常不是技巧复杂，而是边界和预处理别写乱。

你真正要会的是：

- 能先区分这题是在做“纯比较”还是“带一次容错的比较”
- 能把字符过滤和主体逻辑分清

### 3.3 哈希统计 / 分组

适用场景：

- 统计字符出现次数
- 比较两个字符串是否同构 / 异位词
- 按特征分组

代表题：

- `242. 有效的字母异位词`
- `49. 字母异位词分组`
- `205. 同构字符串`

通用套路：

1. 先判断题目更偏：
   - 次数统计
   - 映射关系
   - 分组归类
2. 如果是频次比较，常见主流程是：
   - 统计字符次数
   - 再比较是否一致
3. 如果是分组题，常见主流程是：
   - 为每个字符串构造稳定 key
   - 再用 `HashMap<key, List<String>>` 分桶
4. 如果字符集固定很小，通常优先考虑数组计数，而不是 `HashMap`。

你真正要会的是：

- 能看出“字符串题里的哈希题”本质仍然是计数、映射、分组三种模式
- 能判断什么时候该用数组计数，什么时候该用哈希表

### 3.4 模拟与解析

适用场景：

- 大数相加
- 路径化简
- 字符串转整数
- 压缩 / 展开格式

代表题：

- `415. 字符串相加`
- `8. 字符串转换整数 (atoi)`
- `71. 简化路径`

通用套路：

1. 先把题目拆成若干个确定步骤，不要一上来写大而全的分支。
2. 常见拆法是：
   - 读入
   - 过滤
   - 转换
   - 更新状态
   - 输出结果
3. 如果题目涉及大数、路径、格式解析，通常都要维护一个中间状态：
   - 进位
   - 当前数字
   - 当前路径片段
   - 当前符号
4. 这类题最怕“边解析边胡乱补条件”，更稳的方式是先把状态机或步骤顺序想清楚。

你真正要会的是：

- 先把模拟过程拆成可执行步骤
- 再逐步编码，而不是把所有条件直接塞进一个循环

## 4. 字符串题最容易混淆的概念

- 子串：
  - 连续
- 子序列：
  - 不要求连续
- 前缀：
  - 从开头开始
- 后缀：
  - 到结尾结束
- 回文：
  - 正反读一致

面试里如果这些概念一混，后面的题型判断通常也会跟着错。

## 5. 字符串题的高频坑点

- 没先搞清楚是子串还是子序列
- 频繁字符串拼接导致复杂度变高
- 忽略空串、单字符串、大小写和非字母字符
- 把字符统计题写成重复扫描
- 窗口类题没有明确“什么时候窗口合法”

## 6. 字符串题的面试表达模板

```text
这题输入是字符串，但本质上我会先判断它更像哪一种模式：
如果是连续区间问题，我优先考虑滑动窗口；
如果是从两端向中间判断关系，我会考虑双指针；
如果是字符频次、映射或分组，我会考虑哈希表。
我会先讲清楚题目里的关键定义，比如子串是否要求连续，
再说明状态怎么维护，最后补复杂度和边界。
```

## 8. 你复习字符串时最低要做到什么程度

- 能清楚区分子串和子序列
- 能看出题目到底偏窗口、双指针还是哈希统计
- 能独立写出 `3`、`125`、`242` 这三类基础模板
- 口头表达时，先说“字符串题背后是哪一种模式”，不要只停留在表面

## 9. 本专题练习规划

### 第一阶段：模板入门

1. `125. 验证回文串`
   - 训练重点：
     - 双指针
     - 字符过滤和边界控制
2. `242. 有效的字母异位词`
   - 训练重点：
     - 字符统计
     - 数组计数和哈希计数的选择

### 第二阶段：高频主战

1. `3. 无重复字符的最长子串`
   - 训练重点：
     - 字符串滑动窗口模板
     - 窗口合法性维护
2. `438. 找到字符串中所有字母异位词`
   - 训练重点：
     - 固定长度窗口
     - 频次匹配
3. `415. 字符串相加`
   - 训练重点：
     - 从后往前模拟
     - 进位处理和 `StringBuilder`

### 第三阶段：变体 / 验收

1. `76. 最小覆盖子串`
   - 训练重点：
     - 复杂窗口题
     - 判断自己是否真的掌握窗口收缩条件
2. `8. 字符串转换整数 (atoi)`
   - 训练重点：
     - 多条件模拟
     - 边界处理和代码鲁棒性

## 10. 专题验收标准

- 能先判断题目更偏：
  - 滑动窗口
  - 双指针
  - 哈希统计
  - 字符串模拟
- 能清楚区分子串和子序列，不再混概念
- 能讲清 `3`、`438`、`76` 三题窗口难度为什么逐步升级
- 能做到字符串题不只盯着字符本身，而是先抽象出背后的模式
