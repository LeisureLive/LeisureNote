# 算法专题 08：二分查找

适用目标：

- 建立二分题的统一认知：二分的本质不是“在有序数组里找值”，而是“利用单调性不断缩小答案范围”
- 分清基础二分、边界二分、变体二分、答案空间二分这几类高频模式
- 能把二分题从“会套 `while (left <= right)`”提升到“能解释为什么这里一定能二分、为什么边界这样收缩”
- 减少二分最常见的边界错误、死循环和判定条件写反问题

## 1. 二分题的核心特点

- 二分的前提不是“必须有序”，而是“搜索空间里存在单调性”
- 二分每一轮做的事情只有两步：
  - 选中间位置
  - 根据判定结果砍掉一半不可能的范围
- 真正决定二分能不能写对的，不是模板背得多熟，而是下面三件事是否明确：
  - 当前搜索空间是什么
  - 判定函数是什么
  - 缩边界时保留的是哪一半可行解
- 很多面试中的二分题，难点都不在代码，而在于：
  - 你能不能先识别出单调性
  - 你能不能说清楚“mid 满足条件后为什么往左收还是往右收”

## 2. 什么题应该优先想到二分

如果题目有下面这些信号，通常优先考虑二分：

- 输入本身有序，要求查找某个值或某个边界
- 题目在问：
  - 第一个满足条件的位置
  - 最后一个满足条件的位置
  - 最小可行值 / 最大可行值
- 题目虽然不是直接查数组，但答案具有“可行 / 不可行”的分界
- 题目里出现：
  - “最小速度”
  - “最短时间”
  - “最大容量”
  - “是否可以在某个代价内完成”

如果题目只是普通线性扫描就够，而且没有单调性，通常不该硬上二分。

## 3. 二分题常考模式

看这一节时，建议你每个模式都固定追问三件事：

- 搜索空间到底是什么：下标、值域，还是答案空间
- 判定函数返回 true / false 时，分别说明什么
- 当前这次收缩，是在找“某个值”，还是在找“某条边界”

### 3.1 基础二分：在有序数组里找目标值

适用场景：

- 有序数组中判断目标是否存在
- 返回目标下标
- 不需要找左右边界，只要找到任意一个位置即可

代表题：

- `704. 二分查找`
- `35. 搜索插入位置`

核心理解：

- 这是最基础的二分题，重点是理解二分的基本收缩逻辑
- 当前区间内，如果 `nums[mid]` 已经和目标比较出大小关系，就能直接排除一半区间
- 这类题通常不难，但它是后面所有变体的基础

通用套路：

1. 先确认搜索空间是下标区间 `[left, right]`。
2. 每轮取 `mid`，比较：
   - `nums[mid] == target`
   - `nums[mid] < target`
   - `nums[mid] > target`
3. 如果 `mid` 太小，说明答案只能在右边。
4. 如果 `mid` 太大，说明答案只能在左边。
5. 循环结束后，再根据题意返回：
   - 找到的下标
   - 或插入位置

你真正要会的是：

- 能先说清楚“当前区间里哪一半一定不可能有答案”
- 不是只会背三分支模板

### 3.2 边界二分：找第一个 / 最后一个满足条件的位置

适用场景：

- 找目标值第一次出现的位置
- 找目标值最后一次出现的位置
- 找第一个大于等于某值的位置
- 找最后一个小于等于某值的位置

代表题：

- `34. 在排序数组中查找元素的第一个和最后一个位置`
- `35. 搜索插入位置`
- `278. 第一个错误的版本`

核心理解：

- 这类题不是“找到目标就返回”，而是“找到目标后继续逼近边界”
- 真正要找的不是某个具体元素，而是：
  - 满足条件区间的左边界
  - 或右边界
- 这也是很多人二分最容易写错的地方，因为命中了目标以后，搜索还没结束

通用套路：

1. 先把题目翻译成边界问题：
   - 第一个 `>= target`
   - 最后一个 `<= target`
   - 第一个满足某性质的位置
2. 再决定判定函数：
   - `mid` 满足条件时，边界是不是还可能在左边
   - `mid` 不满足条件时，答案是不是一定在右边
3. 常见主流程是：
   - 如果 `mid` 满足条件，先记录它可能是答案，再继续往更靠边的一侧收缩
   - 如果 `mid` 不满足条件，就把这一侧整体排除
4. 最后返回边界位置，必要时再做一次合法性校验。

你真正要会的是：

- 一看到“第一个 / 最后一个”就直接想到边界二分
- 能讲清为什么命中目标后还要继续缩区间

### 3.3 变体二分：局部有序或结构变化下的查找

适用场景：

- 旋转有序数组
- 山脉数组 / 峰值问题
- 局部单调但不是整体普通有序

代表题：

- `33. 搜索旋转排序数组`
- `153. 寻找旋转排序数组中的最小值`
- `162. 寻找峰值`

核心理解：

- 这类题表面上不是普通有序数组，但仍然存在“每轮至少能确定一半区间性质”的结构
- 比如 `33` 的关键不是背模板，而是：
  - 每次至少有一半区间是有序的
  - 再判断目标是否落在这半边有序区间里
- 所以这类题的本质仍然是利用结构性单调，而不是硬记分类讨论

通用套路：

1. 先问：虽然整体不完全有序，但每轮能不能确定某一半有序。
2. 如果能，就先判断：
   - 左半边有序，还是右半边有序
3. 再判断目标值是否落在这半边有序区间里。
4. 如果落在有序半边，就保留那一边；否则去另一边找。
5. 这类题最重要的不是代码量，而是条件判断方向别写反。

你真正要会的是：

- 能先说出“为什么这里仍然能二分”
- 能区分“普通有序数组二分”和“局部有序结构二分”的判定逻辑差异

### 3.4 答案空间二分：在值域上二分可行解

适用场景：

- 求最小可行值 / 最大可行值
- 题目给的是约束判断，而不是直接查数组
- 可以写出一个“给定答案 x，是否可行”的检查函数

代表题：

- `875. 爱吃香蕉的珂珂`
- `1011. 在 D 天内送达包裹的能力`
- `410. 分割数组的最大值`

核心理解：

- 这类题不是在数组下标上二分，而是在“答案值域”上二分
- 真正的关键不是数据本身有序，而是：
  - 如果某个答案可行
  - 那么比它更大或更小的一侧，往往也整体可行或整体不可行
- 也就是说，可行性在答案空间里形成了分界线

通用套路：

1. 先明确答案范围：
   - 最小可能值是多少
   - 最大可能值是多少
2. 再写检查函数 `check(mid)`：
   - 给定某个候选答案，能不能完成题目要求
3. 如果 `mid` 可行，说明答案可能还能更优，就继续往更优的一侧收缩。
4. 如果 `mid` 不可行，说明答案太激进，需要往另一侧扩大。
5. 这类题最重要的是：
   - `check` 函数必须单调
   - 否则二分没有意义

你真正要会的是：

- 能先把题目翻译成“二分答案 + 可行性检查”
- 能讲清 `mid` 可行时为什么向左收，还是向右收

## 4. 二分题的统一判断框架

做二分题时，可以先按这 4 步判断：

1. 搜索空间是什么：
   - 下标区间
   - 值域区间
   - 答案空间
2. 单调性在哪里：
   - 数组本身有序
   - 某半边一定有序
   - 可行性随答案单调变化
3. `mid` 的判定条件是什么：
   - 和目标值比较
   - 是否满足边界条件
   - 是否可行
4. 收缩规则是什么：
   - `mid` 满足条件后保留左边还是右边
   - `mid` 不满足条件后排除哪一边

如果这 4 件事没想清楚，代码通常会写乱。

## 5. 二分题的高频坑点

- 只会背模板，不先说明搜索空间和判定逻辑
- 把“找值”和“找边界”混在一起
- 命中目标就立刻返回，结果错过边界答案
- `left`、`right` 更新方向写反，导致漏解或死循环
- 变体二分里“哪半边有序”判断错
- 答案空间二分里 `check` 函数不具备单调性
- 循环结束后没有做最终合法性校验

## 6. 二分题的面试表达模板

```text
这题我会先判断它是不是存在单调性，
因为二分的本质不是“数组有序”，而是“搜索空间能被一刀切成一定不可能和可能的两部分”。

如果题目是在有序数组里找位置，我会先判断这是找具体值还是找边界；
如果题目是在求最小可行值，我会把它翻译成答案空间二分，
先定义 check(mid) 表示给定一个候选答案时是否可行，
再根据可行性决定向左收还是向右收。
```

## 8. 你复习二分题时最低要做到什么程度

- 能区分：
  - 基础二分
  - 边界二分
  - 变体二分
  - 答案空间二分
- 能独立写出：
