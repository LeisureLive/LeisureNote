# 算法专题 05：字符串

适用目标：

- 建立字符串题的基础认知：它本质上往往是数组 + 指针 + 哈希的组合
- 区分“子串、子序列、前缀、后缀、回文”这些常见概念
- 避免因为定义不清导致题型识别错误

## 1. 字符串题的核心特点

- 字符串可以看成特殊数组，但更强调字符语义和边界定义
- 很多题的关键不是数据结构本身，而是：
  - 如何维护连续区间
  - 如何判断字符是否重复
  - 如何构造稳定的比较规则
- Java 里字符串不可变，所以频繁拼接时要注意 `StringBuilder`

## 2. 什么题应该优先想到字符串

如果题目有下面这些信号，通常优先考虑字符串思维：

- “最长 / 最短子串”
- “是否为回文”
- “字符串匹配”
- “按字符统计频次”
- “字符串解析 / 模拟”
- “字符串重组 / 反转 / 分组”

## 3. 字符串题常考模式

### 3.1 滑动窗口

适用场景：

- 最长无重复子串
- 包含目标字符集合的最短子串
- 找所有异位词起点

代表题：

- `3. 无重复字符的最长子串`
- `438. 找到字符串中所有字母异位词`
- `76. 最小覆盖子串`

关键点：

- 一定要先分清“窗口是否合法”的定义
- 再决定左指针什么时候缩、右指针什么时候扩

### 3.2 双指针 / 回文判断

适用场景：

- 判断回文串
- 原地反转字符数组
- 从两端向中间收缩

代表题：

- `125. 验证回文串`
- `344. 反转字符串`
- `680. 验证回文串 II`

### 3.3 哈希统计 / 分组

适用场景：

- 统计字符出现次数
- 比较两个字符串是否同构 / 异位词
- 按特征分组

代表题：

- `242. 有效的字母异位词`
- `49. 字母异位词分组`
- `205. 同构字符串`

### 3.4 模拟与解析

适用场景：

- 大数相加
- 路径化简
- 字符串转整数
- 压缩 / 展开格式

代表题：

- `415. 字符串相加`
- `8. 字符串转换整数 (atoi)`
- `71. 简化路径`

## 4. 字符串题最容易混淆的概念

- 子串：
  - 连续
- 子序列：
  - 不要求连续
- 前缀：
  - 从开头开始
- 后缀：
  - 到结尾结束
- 回文：
  - 正反读一致

面试里如果这些概念一混，后面的题型判断通常也会跟着错。

## 5. 字符串题的高频坑点

- 没先搞清楚是子串还是子序列
- 频繁字符串拼接导致复杂度变高
- 忽略空串、单字符串、大小写和非字母字符
- 把字符统计题写成重复扫描
- 窗口类题没有明确“什么时候窗口合法”

## 6. 字符串题的面试表达模板

```text
这题输入是字符串，但本质上我会先判断它更像哪一种模式：
如果是连续区间问题，我优先考虑滑动窗口；
如果是从两端向中间判断关系，我会考虑双指针；
如果是字符频次、映射或分组，我会考虑哈希表。
我会先讲清楚题目里的关键定义，比如子串是否要求连续，
再说明状态怎么维护，最后补复杂度和边界。
```

## 7. 字符串专题建议掌握的代表题

### 入门必做

- `344. 反转字符串`
- `125. 验证回文串`
- `242. 有效的字母异位词`
- `14. 最长公共前缀`

### 面试高频

- `3. 无重复字符的最长子串`
- `438. 找到字符串中所有字母异位词`
- `49. 字母异位词分组`
- `415. 字符串相加`

### 进阶

- `76. 最小覆盖子串`
- `680. 验证回文串 II`
- `8. 字符串转换整数 (atoi)`

## 8. 你复习字符串时最低要做到什么程度

- 能清楚区分子串和子序列
- 能看出题目到底偏窗口、双指针还是哈希统计
- 能独立写出 `3`、`125`、`242` 这三类基础模板
- 口头表达时，先说“字符串题背后是哪一种模式”，不要只停留在表面

## 9. 本专题练习规划

### 第一阶段：模板入门

1. `125. 验证回文串`
   - 训练重点：
     - 双指针
     - 字符过滤和边界控制
2. `242. 有效的字母异位词`
   - 训练重点：
     - 字符统计
     - 数组计数和哈希计数的选择

### 第二阶段：高频主战

1. `3. 无重复字符的最长子串`
   - 训练重点：
     - 字符串滑动窗口模板
     - 窗口合法性维护
2. `438. 找到字符串中所有字母异位词`
   - 训练重点：
     - 固定长度窗口
     - 频次匹配
3. `415. 字符串相加`
   - 训练重点：
     - 从后往前模拟
     - 进位处理和 `StringBuilder`

### 第三阶段：变体 / 验收

1. `76. 最小覆盖子串`
   - 训练重点：
     - 复杂窗口题
     - 判断自己是否真的掌握窗口收缩条件
2. `8. 字符串转换整数 (atoi)`
   - 训练重点：
     - 多条件模拟
     - 边界处理和代码鲁棒性

## 10. 专题验收标准

- 能先判断题目更偏：
  - 滑动窗口
  - 双指针
  - 哈希统计
  - 字符串模拟
- 能清楚区分子串和子序列，不再混概念
- 能讲清 `3`、`438`、`76` 三题窗口难度为什么逐步升级
- 能做到字符串题不只盯着字符本身，而是先抽象出背后的模式
