# ai 基础概念
## 1.LLM
- 大语言模型（Large Language Model）是指能够理解和生成自然语言文本的人工智能模型。
- 目前的 LLM 基本都是通过 Transformer 架构训练出来的

## 2. Token
- Token 是指在自然语言处理中最基本的单元
- Tokenizer 用于文本和 Token 的转换，将输出转为 Token 叫做编码，将 Token 转回文本叫做解码。LLM 只认识 Token，不直接处理文本。
- 1 Token 约等于 1.5 汉字，约等于 0.75 英文单词
- LLM 的输入输出都是 Token，模型的参数也是基于 Token 进行训练的

## 3. Context
- Context 是指模型每次处理任务时接收到的信息总和
- 每次和大模型对话时，输入的文本会被转换成 Token，并与之前的对话历史一起构成当前的 Context，模型会基于这个 Context 来生成回答
- Context 包含了对话历史、用户问题、当前输出、工具列表、System Prompt 等信息
- Context window: 即大模型 Context 最多能存储的 Token 数量，目前已经达到了 100w Token 以上

## 4. RAG
- RAG（Retrieval-Augmented Generation）是一种结合了检索和生成的技术，能够让模型在生成文本时参考外部知识库中的信息
- RAG 的核心思想是先通过检索模块从知识库中找到与用户问题相关的文档或信息，然后将这些信息作为 Context 输入到生成模块中，让模型在生成回答时能够参考这些外部知识，从而提高回答的准确性和丰富性
- RAG 可以帮助模型克服知识截止日期的问题，让模型能够利用最新的信息来回答用户的问题，同时也能够提高模型在特定领域的表现，因为模型可以参考领域相关的知识库来生成回答
- RAG 的实现通常包括以下几个步骤：
  1. 用户输入问题
  2. 检索模块从知识库中检索相关文档或信息
  3. 将检索到的信息与用户问题一起构成 Context
  4. 生成模块基于这个 Context 来生成回答
- RAG 技术可以帮助减小模型的参数规模，因为模型不需要记住所有的知识，而是可以通过检索来获取需要的信息，从而提高模型的效率和性能


## 5. Prompt
- Prompt 是指在与大模型交互时，用户输入的文本或指令，用于引导模型生成回答
- Prompt Engineering 是指设计和优化 Prompt 的过程，目的是让模型能够更好地理解用户的意图，从而生成更准确和有用的回答
- Prompt 分类：
  1. System Prompt：用于设定模型的行为和角色，例如“你是一个专业的医生，请回答以下问题”
  2. User Prompt：用户输入的具体问题或指令，例如“什么是 RAG？”
  3. Tool Prompt：用于引导模型使用特定工具或接口，例如“请调用天气查询 API 获取今天的天气信息”

## 6. Tool
- Tool 是指在与大模型交互时，大模型感知和影响外部环境的函数
- Tool 的实现通常包括以下几个步骤：
  1. 用户输入问题
  2. 模型识别出需要调用工具来获取信息或执行操作
  3. 模型生成调用工具的指令或请求
  4. 工具执行相应的操作并返回结果
  5. 模型将工具返回的结果作为 Context 来生成最终的回答
- 模型不直接执行工具，而是生成调用工具的指令或请求，由外部系统来执行工具并返回结果，这样可以保证模型的安全性和可控性，同时也能够让模型利用各种外部资源来增强自己的能力
- 执行工具的角色通常由一个叫做 Agent 的组件来负责，Agent 会根据模型生成的指令来调用相应的工具，并将工具的结果返回给模型，让模型能够基于这些结果来生成更准确和有用的回答

## 9. Agent Skill
- 可以理解为特定场景给 Agent 看的说明文档，包含了触发场景、需要的工具、做事步骤和规则、输入输出格式、示例等信息
- 渐进式披露机制：根据对话进展逐步披露提示词模版的内容，避免一次性给模型过多信息导致混乱，同时也能够让模型更好地理解当前的任务和需要调用的工具，从而提高回答的准确性和相关性
