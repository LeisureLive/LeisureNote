# 算法专题 10：图

适用目标：

- 建立图题的统一认知：图题的关键不是“多背几道 BFS / DFS”，而是先判断题目在问连通性、最短步数，还是依赖顺序
- 分清网格搜索、普通图遍历、多源 / 分层 BFS、拓扑排序这几类高频模式
- 能把图题从“会写 `visited`”提升到“能解释为什么这里要用 DFS / BFS、图该怎么建、状态为什么不会重复搜”
- 减少图题最常见的重复访问、层数统计错误、拓扑排序入度维护错误问题

## 1. 图题的核心特点

- 图题的本质是：
  - 点和点之间存在连接关系
  - 你要在这种连接关系上做遍历、搜索或排序
- 图题最重要的不是 API，而是先看清下面三件事：
  - 节点是什么
  - 边是什么
  - 题目到底在问“能不能到”“最少几步到”，还是“先后依赖顺序”
- 很多题表面长得不一样，但本质上都落在几类问题里：
  - 找连通块
  - 找最短路径 / 最少步数
  - 按依赖关系排序
  - 在状态空间里做搜索
- 真正决定图题能不能做对的，通常不是代码量，而是：
  - 图有没有建对
  - `visited` 在防什么重复
  - 队列里的“这一层”到底代表什么语义

## 2. 什么题应该优先想到图

如果题目有下面这些信号，通常优先往图的方向想：

- 输入本身就是：
  - 邻接关系
  - 课程依赖
  - 节点和边
  - 字符串变换关系
- 输入虽然不是显式图，但本质上是图：
  - 网格里的上下左右移动
  - 单词从一个状态变到另一个状态
  - 每个状态可以扩展出下一批状态
- 题目在问：
  - 一共有多少块
  - 某个点能不能到另一个点
  - 从起点到终点最少几步
  - 有没有环
  - 是否存在合法执行顺序

如果题目输入是树，优先按树题看；如果题目只是在枚举所有方案，更可能属于回溯；如果题目在求最优值且状态可复用，才更可能是 DP。

## 3. 图题常考模式

看这一节时，建议你每个模式都固定追问三件事：

- 节点是什么，边怎么定义
- 题目更像“遍历所有可达点”，还是“按层扩展找最短步数”
- 重复访问来自哪里，要靠什么状态剪掉

### 3.1 连通性：网格 / 图上的 DFS 与 BFS

适用场景：

- 求岛屿数量
- 求最大连通块面积
- 判断某点是否可达
- 遍历某个连通分量内的所有节点

代表题：

- `200. 岛屿数量`
- `695. 岛屿的最大面积`
- `733. 图像渲染`

核心理解：

- 这类题的重点不是“最短路径”，而是把一个连通块完整走完
- 你每发现一个还没访问过的起点，就说明找到了一个新的连通块
- 接下来无论用 DFS 还是 BFS，本质都在做同一件事：
  - 把和它连通的所有节点都标记掉

通用套路：

1. 先确定邻接关系：
   - 网格题常见是上下左右四个方向
   - 普通图题常见是邻接表
2. 外层遍历所有节点或网格位置。
3. 发现一个满足条件且未访问过的节点，就启动一次 DFS / BFS。
4. 在这次搜索里，把整个连通块都标记为已访问。
5. 外层继续扫描时，就不会重复计数。

你真正要会的是：

- 能说清 `visited` 或原地改值是在防“重复计数”和“死循环”
- 能解释为什么这类题 DFS 和 BFS 都能做，只是写法不同

### 3.2 最短步数：分层 BFS 与多源 BFS

适用场景：

- 无权图里求最少步数
- 每走一步代价相同
- 问题天然是一层一层扩展
- 有多个起点同时向外扩散

代表题：

- `994. 腐烂的橘子`
- `127. 单词接龙`
- `1091. 二进制矩阵中的最短路径`

核心理解：

- BFS 在无权图里之所以适合求最短步数，是因为它天然按层扩展
- 先到达某个节点的那一层，就是从起点到它的最短步数
- 多源 BFS 的本质不是更复杂，而是：
  - 把所有起点同时放进队列作为第 0 层
  - 让扩散一起发生

通用套路：

1. 先确认题目是不是在问“最少几步 / 最短转换次数”。
2. 如果边权都相同，优先考虑 BFS。
3. 如果有多个天然起点，就把所有起点一起入队。
4. 每轮处理一整层节点，再把下一层候选入队。
5. 通过层数统计、步数数组，或节点状态里自带距离来记录答案。

你真正要会的是：

- 能解释为什么“按层扩展”就对应“最短步数”
- 能说清多源 BFS 不是多个 BFS 叠加，而是把多个起点视为同一层起点

### 3.3 依赖顺序：拓扑排序与环检测

适用场景：

- 课程依赖
- 任务编排
- 先后顺序约束
- 判断是否存在环

代表题：

- `207. 课程表`
- `210. 课程表 II`
- `1462. 课程表 IV`

核心理解：

- 这类题的重点不是“从某个点走多远”，而是“有没有合法顺序”
- 只要图里有环，就不存在完整的合法执行顺序
- 拓扑排序最常见的做法是：
  - 统计每个节点入度
  - 把当前入度为 0 的节点先处理掉
  - 再逐步削减它影响到的后继节点入度

通用套路：

1. 先把依赖关系建成邻接表。
2. 同步统计每个节点的入度。
3. 把所有入度为 0 的节点入队。
4. 每次弹出一个节点，就相当于把它从图里“删除”。
5. 更新它所有后继节点的入度，新的入度为 0 节点继续入队。
6. 最后看被处理的节点数是否等于总节点数：
   - 相等：无环
   - 不相等：有环

你真正要会的是：

- 能说清“入度为 0”为什么代表当前没有前置依赖
- 能解释为什么“处理节点数不到总数”就说明图里有环

### 3.4 建图与状态搜索：图题真正的入口常常在“抽象”

适用场景：

- 单词接龙
- 状态变换
- 题面没直接给图，但可以抽象成状态图

代表题：

- `127. 单词接龙`
- `752. 打开转盘锁`
- `433. 最小基因变化`

核心理解：

- 这类题的难点通常不是 DFS / BFS 模板，而是先把“状态”和“邻接关系”抽象出来
- 例如单词接龙里，节点不是字符本身，而是一个完整单词
- 两个节点之间有没有边，取决于它们是否能一步变换

通用套路：

1. 先定义节点：
   - 一个单词
   - 一个密码状态
   - 一个坐标位置
2. 再定义一步操作会产生哪些相邻状态。
3. 然后再决定用 DFS、BFS 还是其他方法。
4. 只要是“最少几步变到目标状态”，优先考虑 BFS。

你真正要会的是：

- 先把题目翻译成“状态图”，再写搜索
- 不要还没建模就直接硬套遍历模板

## 4. 图题的统一判断框架

做图题时，可以先按这 5 步判断：

1. 节点是什么：
   - 坐标
   - 单词
   - 课程
   - 状态
2. 边是什么：
   - 上下左右移动
   - 依赖关系
   - 一步合法变换
3. 题目到底在问什么：
   - 连通性
   - 最短步数
   - 合法顺序
4. 该用什么搜索：
   - DFS
   - BFS
   - 多源 BFS
   - 拓扑排序
5. 重复状态怎么防：
   - `visited`
   - 原地修改
   - 入度数组

如果这 5 件事没想清楚，图题通常会在“建图”和“状态控制”上写乱。

## 5. 图题的高频坑点

- 没先定义节点和边，就急着套 DFS / BFS
- `visited` 标记时机不对，导致重复入队或死循环
- BFS 里层数统计混乱，结果最短步数多算或少算
- 多源 BFS 只放了一个起点，导致语义错误
- 网格题忘记处理边界判断
- 拓扑排序没同时维护邻接表和入度
- 判断有环时只看队列是否空，没看最终处理节点数

## 6. 图题的面试表达模板

```text
这题我会先把它抽象成图问题，先明确节点和边分别是什么。

如果题目是在问连通块数量或某个区域是否可达，我会优先考虑 DFS / BFS 做遍历；
如果题目是在问最少几步，而且每一步代价相同，我会优先考虑按层扩展的 BFS；
如果题目是在问课程依赖或执行顺序，我会把它看成有向图的拓扑排序问题，
用入度数组加队列判断是否有环，并构造合法顺序。

接下来我会再补 visited 的设计，以及答案到底是在“遍历次数”“层数”还是“处理节点顺序”里产生。
```

## 8. 你复习图题时最低要做到什么程度

- 能区分：
  - 连通性问题
  - 最短步数问题
  - 拓扑排序问题
- 能独立写出：
  - `200` 的网格 DFS / BFS
  - `994` 的多源 BFS
  - `207` 的拓扑排序
- 能讲清：
  - DFS 和 BFS 在图题里分别更适合什么
  - 为什么 BFS 适合无权最短路
  - 入度数组在拓扑排序里控制什么
- 口头表达时，先说：
  - 节点和边怎么定义
  - 搜索方式为什么这样选
  - `visited` 或入度在防什么错误
