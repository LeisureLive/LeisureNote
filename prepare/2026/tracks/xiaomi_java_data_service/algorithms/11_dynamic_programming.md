# 算法专题 11：动态规划

适用目标：

- 建立 DP 题的统一认知：DP 的关键不是“背转移方程”，而是先判断问题能不能拆成可复用的子问题
- 分清一维线性 DP、完全背包、序列型 DP 这几类当前最值得拿稳的高频模式
- 能把 DP 题从“看答案抄公式”提升到“能解释状态、转移、初始化、遍历顺序为什么这样定义”
- 减少 DP 最常见的状态定义模糊、初始化错误、遍历顺序写反问题

## 1. 动态规划的核心特点

- DP 适合的问题通常有两个共同点：
  - 最优子结构
  - 重叠子问题
- 直白一点说，就是：
  - 当前答案依赖于更小规模子问题的答案
  - 这些子问题会被反复用到
- DP 最重要的不是“公式好不好背”，而是先看清下面四件事：
  - 状态是什么
  - 状态转移方程是什么
  - 初始条件是什么
  - 遍历顺序为什么成立
- 真正决定 DP 能不能写对的，通常不是代码量，而是：
  - 你有没有把问题压成一个稳定状态
  - 你知不知道当前状态依赖哪些更早状态

## 2. 什么题应该优先想到 DP

如果题目有下面这些信号，通常优先考虑 DP：

- 问最优值：
  - 最少步数
  - 最大收益
  - 最长长度
  - 最小代价
- 当前决策依赖前面若干步的结果
- 暴力递归会重复计算大量相同子问题
- 题目可以描述成：
  - 到位置 `i` 为止的最优答案
  - 凑出容量 `j` 的最优答案
  - 以 `i` 结尾的最优答案

如果题目要求列出所有方案，更可能是回溯；如果题目在图上求最短步数，很多时候先考虑 BFS；如果题目只需局部最优选择即可推进，可能是贪心。

## 3. DP 题常考模式

看这一节时，建议你每个模式都固定追问四件事：

- `dp[i]` 或 `dp[i][j]` 到底表示什么
- 当前状态依赖哪些更小状态
- 初始值怎么设
- 为什么这个遍历顺序不会读到“未来”的状态

### 3.1 一维线性 DP：当前位置的答案依赖前几个位置

适用场景：

- 爬楼梯
- 打家劫舍
- 最小花费爬楼梯
- 斐波那契型递推

代表题：

- `70. 爬楼梯`
- `198. 打家劫舍`
- `746. 使用最小花费爬楼梯`

核心理解：

- 这类题的主线通常是：
  - 走到第 `i` 个位置时，最优答案怎么由前面若干个位置推出来
- `70` 的重点是“方案数递推”
- `198` 的重点是“当前位置选 or 不选”的最优决策
- 它们看起来不一样，但本质都在做：
  - 先定义一维状态
  - 再从前往后推

通用套路：

1. 先定义 `dp[i]` 的含义：
   - 到第 `i` 级台阶的方法数
   - 到第 `i` 个房子为止能偷到的最大金额
2. 再写转移：
   - `70`：来自 `i - 1` 和 `i - 2`
   - `198`：来自“不偷当前”和“偷当前”两种选择
3. 再处理初始条件：
   - `dp[0]`、`dp[1]` 具体含义要和状态定义一致
4. 最后确定遍历方向：
   - 通常从小到大

你真正要会的是：

- 能先说出 `dp[i]` 的自然语义，再写公式
- 能解释 `198` 为什么不是简单斐波那契，而是“选 / 不选”决策

### 3.2 完全背包 / 组合最值：容量维度上的 DP

适用场景：

- 凑金额
- 求最少硬币数
- 求组合数
- 一个物品可重复使用

代表题：

- `322. 零钱兑换`
- `518. 零钱兑换 II`
- `279. 完全平方数`

核心理解：

- 这类题常见的难点不在“会不会写数组”，而在于：
  - 当前 `dp[j]` 表示最少值、最大值，还是方案数
  - 题目是在求排列数还是组合数
  - 物品能不能重复使用
- `322` 的重点是最少硬币数，属于“最值型完全背包”
- `518` 的重点是方案数，属于“计数型完全背包”

通用套路：

1. 先定义 `dp[j]`：
   - 凑出金额 `j` 的最少硬币数
   - 凑出金额 `j` 的方案数
2. 再决定初始值：
   - 最值题常用一个“不可能大值”做初始化
   - 计数题常有 `dp[0] = 1`
3. 再确定转移：
   - 最值题通常是 `dp[j] = min(dp[j], dp[j - coin] + 1)`
   - 计数题通常是累加
4. 最后确认循环顺序：
   - 完全背包常见是容量正序
   - 但组合数和排列数的外内层循环含义不同

你真正要会的是：

- 能区分“求最少个数”和“求有多少种方法”不是同一类转移
- 能解释为什么 `dp[0] = 1` 在计数题里代表“凑出 0 的空方案”

### 3.3 序列型 DP：以当前位置结尾的最优答案

适用场景：

- 最长递增子序列
- 最长公共子序列
- 最长回文子序列
- 编辑距离

代表题：

- `300. 最长递增子序列`
- `1143. 最长公共子序列`
- `72. 编辑距离`

核心理解：

- 这类题的主线通常是：
  - 以当前位置结尾
  - 或前 `i` 个、前 `j` 个元素范围内
  - 最优答案是多少
- `300` 最值得拿稳，因为它非常适合练“状态含义”和“枚举前置状态”
- 面试里不一定要求你把所有二维 DP 都吃透，但至少要能稳住 `300` 这种一维序列 DP

通用套路：

1. 先定义状态：
   - `dp[i]` 表示以 `nums[i]` 结尾的 LIS 长度
   - 或 `dp[i][j]` 表示前 `i`、前 `j` 范围内的最优值
2. 再看当前状态由哪些更早状态转移而来。
3. 如果当前元素能接在前一个状态后面，就尝试更新最优值。
4. 最后从所有状态里取全局最优答案，或直接读表尾。

你真正要会的是：

- 能解释为什么 `300` 要定义成“以 `i` 结尾”，而不是“前 `i` 个数的答案”
- 能说清“局部状态”和“全局答案”有时不是同一个位置

## 4. DP 入门四问

做 DP 题时，建议先固定问自己这四句：

1. 状态是什么：
   - `dp[i]`
   - `dp[i][j]`
2. 转移方程是什么：
   - 当前状态由哪些更小状态推出来
3. 初始条件是什么：
   - `dp[0]`、第一行、第一列怎么设
4. 遍历顺序是什么：
   - 从前往后
   - 从小到大
   - 还是按区间长度

如果这四件事没想清楚，DP 代码通常只是“看起来像公式”，但其实并不稳定。

## 5. DP 题的统一判断框架

做 DP 题时，可以先按这 5 步判断：

1. 题目求的是什么：
   - 最值
   - 方案数
   - 可行性
2. 状态怎么压：
   - 位置
   - 容量
   - 子序列范围
3. 当前状态依赖谁：
   - 前一个
   - 前两个
   - 更早一批状态
4. 初始条件怎么设：
   - 哪些状态天然已知
5. 遍历顺序为什么这样写：
   - 保证转移时依赖项已经算出来

## 6. DP 题的高频坑点

- 上来就背公式，没有先定义状态语义
- `dp[i]` 含义模糊，写着写着变了
- 初始化和状态定义不一致
- 把“方案数”和“最值”混成同一种转移
- 背包题循环顺序写反
- 只会抄答案，不会解释为什么依赖这些前置状态
- 结果应该取 `dp[n]` 还是取所有状态最大值，没有分清

## 7. DP 题的面试表达模板

```text
这题我会先判断它是不是一个可以复用子问题答案的最优问题，
如果是，我会优先考虑动态规划。

我先定义状态，说明 dp 数组里每个位置到底代表什么；
再写出当前状态如何由更小状态转移而来；
然后补初始化和遍历顺序，
因为 DP 真正稳定的关键不是公式本身，而是状态语义和依赖方向必须一致。

如果题目是在求最少代价、最大收益，我会按最值型 DP 去想；
如果是在求方案数，我会特别注意初始化和循环顺序是否改变了语义。
```

## 9. 你复习 DP 时最低要做到什么程度

- 能区分：
  - 一维线性 DP
  - 完全背包 / 容量型 DP
  - 序列型 DP
- 能独立写出：
  - `70` 的入门模板
  - `198` 的一维最值 DP
  - `322` 的完全背包最值题
- 能讲清：
  - DP 入门四问分别在解决什么
  - `322` 和 `518` 为什么不能写成同一种转移
  - `300` 为什么常定义成“以当前位置结尾”
- 口头表达时，先说：
  - 状态定义
  - 转移来源
  - 初始化
  - 遍历顺序
