# 小米相关岗位合并要求与准备重点

适用范围：

- 小米 Java 研发工程师（数据服务）
- 小米数据平台开发工程师 Java 方向
- 小米数据工场 / 数据平台后端研发方向

说明：

- 这份文档不是原始 JD 存档，而是基于当前已收集的 3 个岗位要求做的合并提炼。
- 目标不是逐条复述招聘文案，而是输出后续能直接驱动知识地图、冲刺计划和模拟面试的统一准备基线。
- 后续如果补充新的岗位要求，这份文档应优先更新，再反向调整知识地图和月计划。

## 1. 合并后的岗位定位

这几类岗位虽然名称不同，但本质上都落在同一条主线：

- 主体岗位还是 Java 后端研发。
- 业务语境偏数据服务、数据平台、数据工场、BI 支撑。
- 目标级别偏资深，要求不只是“能开发功能”，而是：
  - 能做模块设计
  - 能做系统架构和调优
  - 能处理复杂链路和线上问题
  - 能和产品、数据、前端、测试协作推进项目

所以后续准备不能只按“纯 Java 八股”来做，也不能只按“纯大数据开发”来做，而要按下面这条组合能力去准备：

- Java 后端核心能力
- 分布式系统设计与调优能力
- 数据平台 / 数据服务场景理解
- 工程质量和跨团队协作能力

## 2. 合并后的岗位要求

### 2.1 必须拿稳的共性硬要求

#### A. Java 核心基础

- 精通 Java 语言
- 多线程编程
- JVM 原理与调优
- 常用设计模式
- Java Web 基础
- 基本算法与数据结构

这部分不是只要求“知道概念”，而是要求：

- 能解释机制
- 能结合线上问题说取舍
- 能把它们落到系统设计和项目实现里

#### B. 主流 Java 框架链

- Spring
- Spring MVC
- Spring Boot
- MyBatis
- Dubbo
- Spring Cloud

这里的要求不是简单会用，而是：

- 能讲主流程
- 能讲关键机制
- 能讲常见坑
- 能结合真实项目说明为什么这么选

#### C. 分布式与高可用能力

- RPC
- 缓存
- 消息队列
- 负载均衡
- 分布式事务
- 高性能 / 高并发 / 高可用设计
- 系统调优和优化

这一块是资深岗的核心分水岭。

面试官要看的不是：

- 你知道多少中间件名词

而是：

- 你能不能把这些组件放进一条完整系统链路里
- 你能不能说清它们分别在解决什么问题
- 你知不知道性能、可靠性、一致性之间的取舍

#### D. 数据库与数据引擎能力

- SQL
- MySQL
- Redis
- MongoDB
- SQLServer
- Doris
- Holo

这里的要求已经明显超过“只会 MySQL”。

更准确的准备方式是：

- 先拿稳 SQL / MySQL / Redis 这条主线
- 再补多数据库 / 多引擎的使用边界、优缺点和场景选型

面试里更可能问到的是：

- 为什么这里用 MySQL，不用 Redis / MongoDB
- 为什么这个查询更适合 Doris / Holo 这类分析引擎
- OLTP 和 OLAP 在这类业务里怎么分工

#### E. 数据平台 / 大数据相关能力

- Hadoop
- Spark
- Flink
- Presto
- Kafka
- Doris
- ElasticSearch

这一块在不同岗位里的深浅不完全一样，但共同点很明确：

- 你要能理解数据链路
- 你要能理解平台化设计
- 你要能解释不同组件在链路中的角色

如果岗位更偏数据平台，面试官通常还会继续看：

- 是否有数据中台 / 数据工场 / 数据开发平台经验
- 是否能从“数据研发效率”和“平台能力抽象”角度思考设计

### 2.2 明显加分项

#### A. BI / 报表平台经验

- Tableau
- Power BI
- 神策
- Superset
- Metabase

这类内容更像：

- 你对上层数据产品和分析场景有没有实际感知

不是当前主线，但如果你有相关经历，可以作为强加分点使用。

#### B. 数据平台业务经验

- DMP
- AB 实验平台
- 一站式数据开发平台
- 数据中台核心产品

这类经验的价值在于：

- 它比“普通业务后端”更贴近岗位语境
- 很适合用来打项目面和系统设计题

#### C. 工程经验加分项

- 架构优化
- 系统重构
- 数据库调优
- 大数据开发
- 测试用例和质量保障

这些点对资深岗很重要，因为它们直接体现：

- 你不是只会写功能
- 你还能维护复杂系统、改善质量和稳定性

#### D. 协作拓展加分项

- Vue 等前端框架
- 和产品、前端、数据、测试紧密协作的经验

这类内容一般不是决定性硬要求，但在数据平台或 BI 相关场景里会比普通后端岗位更有价值。

### 2.3 软性能力要求

这几个 JD 里虽然表达方式不同，但软性要求非常一致：

- 独立分析和解决问题能力
- 良好的沟通表达能力
- 跨团队协作能力
- 自驱动和责任心
- 对复杂问题保持持续推进能力
- 能站在业务方 / 数据研发 / 产品等不同角色角度思考问题

这意味着后续准备不能只做知识复习，还必须补：

- 项目表达
- 架构表达
- 排障表达
- 协作与行为面表达

## 3. 这些岗位真正最看重什么

如果把 3 个岗位 JD 压缩成几个最关键的判断，实际最看重的是下面 5 件事：

### 3.1 你是不是一个“能打硬仗的 Java 后端”

也就是：

- Java 基础稳不稳
- 并发 / JVM / SQL / 中间件能不能承接深挖
- 框架链能不能讲清主流程和实战问题

### 3.2 你有没有“分布式系统和数据服务”的工程判断力

也就是：

- 能不能做高并发和高可用设计
- 能不能讲系统瓶颈、限流、降级、扩展、容灾
- 能不能解释缓存、MQ、RPC、事务这些东西怎么配合

### 3.3 你有没有“数据平台 / 数据链路”语境

也就是：

- 不只是做普通 CRUD
- 而是要能理解：
  - 数据接入
  - 数据处理
  - 数据订阅
  - 实时查询
  - 数据开发平台
  - 数据分析引擎

### 3.4 你能不能承担资深岗的设计和排障责任

也就是：

- 系统模块设计
- 架构选型
- 复杂问题分析
- 重点项目技术支持
- 线上问题排查和止损

### 3.5 你能不能把真实经验讲得像“岗位匹配”

也就是：

- 不只是说自己做了什么
- 而是能说清：
  - 为什么这么设计
  - 为什么这么取舍
  - 为什么这能支撑数据服务 / 数据平台场景

## 4. 合并后的准备重点

### 4.1 P0：必须优先拿稳

这些是所有岗位共同的硬核心，不拿稳后面很难打。

- Java 并发
- JVM
- MySQL / SQL
- Redis
- Kafka
- Spring 核心
- Spring Boot / Spring MVC
- MyBatis
- Dubbo / RPC 基础

这里的要求不是：

- 每题都背成长答案

而是：

- 至少能稳定答到 3 分钟版本
- 高频题能承接 1 到 2 轮追问

### 4.2 P1：资深岗和数据平台岗的关键增量

- 系统设计
- 高性能 / 高并发 / 高可用
- 缓存、MQ、负载均衡、分布式事务
- 数据平台 / 数据服务场景设计
- Flink / Spark / Doris / Holo / Presto / ElasticSearch 的角色理解
- 多数据库 / 多引擎选型能力
- Linux / 网络 / 排障

这部分是最容易决定面试上限的内容。

如果 P0 决定你“能不能进入有效对话”，P1 通常决定你“能不能像资深候选人”。

### 4.3 P2：项目与表达层

- 项目介绍主线
- 项目亮点提炼
- 难点复盘
- 架构升级表达
- 跨团队协作表达
- 线上问题处理案例

这部分的重要性不低，尤其你本身有数据平台背景，项目面是很大的胜负手。

### 4.4 P3：加分项

- BI / 报表平台
- DMP / AB 实验平台
- Vue 等前端协作能力
- 测试用例和质量保障表达

这些内容可以补，但不应该抢 P0 / P1 主线时间。

## 5. 对你当前情况的直接影响

基于你现在的背景，这些岗位要求落到准备动作上，最重要的是下面几条。

### 5.1 你现有优势要继续放大

- 基础数据平台背景
- 数据接入 / 处理 / 订阅 / 查询经验
- Kafka、Redis、MySQL、Flink、Hive、Impala 等数据相关技术栈
- 有真实性能优化和横向扩展经验

这些是你和普通 Java 后端候选人相比最有差异化的部分。

### 5.2 你当前最需要补的短板更明确了

- Spring / Spring MVC / Spring Boot / MyBatis / Dubbo / Spring Cloud
- 分布式基础的系统化表达：
  - RPC
  - 缓存
  - MQ
  - 负载均衡
  - 分布式事务
- 多数据库 / 多引擎选型表达
- 数据平台化设计表达
- Linux / Java Web / 设计模式 / 测试质量意识

### 5.3 你后续不该平均补所有东西

更合理的顺序应该是：

1. 先拿稳 Java 后端主线硬要求。
2. 再补资深岗系统设计和数据平台语境。
3. 最后视时间补 BI / 前端 / DMP / AB 实验平台这类加分项。

## 6. 当前最推荐的准备重点版本

如果只看接下来 3 到 4 周，最推荐的准备重点排序是：

1. Java 并发、JVM、MySQL、Redis、Kafka 高频题稳定答题
2. Spring 全家桶、MyBatis、Dubbo / RPC 主流程和常见坑
3. 系统设计：
   - 高并发
   - 高可用
   - 缓存
   - MQ
   - 分布式事务
   - 数据服务链路
4. 数据平台语境：
   - Flink / Spark / Doris / Holo / Presto / ES 的角色分工
   - 一站式数据开发平台的能力抽象
5. 项目面打磨：
   - 实时数据订阅 + 实时查询
   - id-mapping
6. 排障与性能优化案例

## 7. 后续怎么用这份文档

后续每次新增岗位要求或调整主线时，建议固定按下面顺序处理：

1. 先更新这份文档：
   - 哪些是新增硬要求
   - 哪些只是加分项
2. 再调整知识地图：
   - 模块优先级
   - 目标深度
3. 再调整 1 个月计划：
   - 每周目标
   - 每日安排
4. 最后才调整专题复习和模拟面试题单

结论：

- 这几份小米岗位要求可以合并成一条统一准备主线
- 主线核心不是“背更多名词”，而是“Java 后端 + 分布式 + 数据平台语境 + 资深岗表达”
- 后续知识地图和月计划都应该以这份合并要求为上游依据动态调整
